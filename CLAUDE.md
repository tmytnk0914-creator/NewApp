# CLAUDE.md

このファイルはClaude Codeがこのプロジェクトで作業する際のガイドラインを定義します。

## プロジェクト概要

新規アプリケーション開発プロジェクト。

## Git運用ルール

### 基本方針
- **コードを変更するたびに必ずGitHubへプッシュすること**
- コミットは意味のある単位でまとめ、何を変更したか明確なメッセージを書く
- `main` ブランチへの直接プッシュは避け、原則としてfeatureブランチを使用する

### コミット・プッシュの手順
コードを変更した後は毎回以下の手順を実行する：

```bash
git add <変更ファイル>
git commit -m "変更内容の要約"
git push origin <ブランチ名>
```

### コミットメッセージ規則
形式: `<種別>: <要約>`

| 種別 | 用途 |
|------|------|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `refactor` | リファクタリング |
| `docs` | ドキュメント更新 |
| `test` | テスト追加・修正 |
| `chore` | ビルド・設定変更 |

例: `feat: ユーザー認証機能を追加`

### ブランチ命名規則
- 機能追加: `feature/<機能名>`
- バグ修正: `fix/<バグ概要>`
- リリース: `release/<バージョン>`

## 開発ルール

- コードを変更するたびにプッシュする（作業途中でも `WIP:` プレフィックスを付けてプッシュ可）
- PRはレビュー後にマージする
- テストが存在する場合は変更前後でテストが通ることを確認する

## 注意事項

- `.env` ファイルや秘密鍵はコミットしない（`.gitignore` に追加する）
- 大きなバイナリファイルはGit LFSを使用する
